Meta Ads / AI Business Assistant

Turning signal health into guided data connection workflows

  • AI Product
  • Signals Workflow
  • Recommendation UX
Role
Designed the signals flow and interaction model for AI-guided advertiser recommendations, translating signal diagnostics into explainable guidance and decision-support patterns.
Design decision
Turn invisible signal gaps into guided setup paths that advertisers could understand, trust, and complete.
Impact
Supported Conversions API adoption, Meta Pixel growth, event coverage improvements, and partner integration recommendations. Public beta cited 20% higher common-account-issue resolution and 12% lower cost per result.
01 — Context

Advertisers depend on signals they often cannot see.

Advertisers rely on high-quality signals to help Meta understand what happens after an ad click or view: purchases, leads, subscriptions, and other business events.

But signal setup is fragmented. Some advertisers use Meta Pixel. Some need Conversions API. Some rely on partner platforms. Others have incomplete event coverage or disconnected data sources.

The product challenge was not simply telling advertisers to improve signals. It was helping them understand what was missing, why it mattered, and which setup path was most appropriate for their business.

02 — Problem

Signal quality is invisible until something breaks.

Advertisers often saw recommendations related to Conversions API adoption, event coverage, or Pixel growth, but the underlying logic was hard to understand.

Question 01 Which signal is missing?
Question 02 Why does it matter?
Question 03 Which setup path should I trust?
03 — Insight

Advertisers were not trying to debug a data pipeline.

They were trying to answer a business question: is Meta receiving the right data to optimize my campaigns?

If the user has to understand the data pipeline before they can fix the signal, the workflow is doing the system's translation work too late.
04 — Model

From diagnostics to guided setup.

05 — Smart Partner Integration

Recommend the path, not just the tool.

One highlight was Meta Smart Partner Integration: a recommendation engine that identified the most relevant partner integration method for an advertiser to connect their data to Meta.

Rather than asking advertisers to choose from a generic list of integration options, the system could recommend a path based on current signal setup, business context, and available partner connections.

06 — One-click Installation

Speed only works when users understand what is happening.

For eligible advertisers, one-click installation reduced setup friction by turning a technical integration process into a guided confirmation flow.

The experience had to balance speed with trust. A one-click action is only helpful if users understand what is being connected and why the system recommends it.

07 — Principles

Make signal recommendations inspectable before they become actionable.

01 Explain the signal first

A setup recommendation should start with the business signal it improves.

02 Recommend the path

Advertisers should not need to choose between Pixel, Conversions API, and partner setup from scratch.

03 Make setup inspectable

Even when installation is simplified, users should understand what data connection is being created.

04 Close the loop

After setup, the product should help advertisers understand whether signal coverage improved.

08 — Reflection

A good AI assistant should not make users think like the system.

This work taught me that AI recommendations are only as useful as the workflow around them.

For signal products, the hardest design problem is not showing advertisers more diagnostics. It is translating invisible backend conditions into a path they can understand, trust, and complete.

A good AI assistant helps the system speak in the user's decision language.
